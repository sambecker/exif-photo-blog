# Agent Context

This document provides context for AI agents working on the exif-photo-blog project.

## Project Overview

A Next.js photo blog application that stores photos with original EXIF camera data. Supports multiple storage providers and can be self-hosted.

## Key Technologies

- **Framework:** Next.js 16 (App Router)
- **Language:** TypeScript
- **Package Manager:** pnpm
- **Database:** PostgreSQL
- **Storage:** Vercel Blob, Cloudflare R2, AWS S3, MinIO
- **Authentication:** Auth.js v5 (next-auth)
- **Image Processing:** sharp, next/image

## Storage Configuration

The project supports multiple storage providers. Only one can be active at a time.

### Environment Variables

| Provider | Required Variables |
|----------|-------------------|
| Vercel Blob | `BLOB_READ_WRITE_TOKEN` |
| Cloudflare R2 | `CLOUDFLARE_R2_*`, `NEXT_PUBLIC_CLOUDFLARE_R2_*` |
| AWS S3 | `AWS_S3_*`, `NEXT_PUBLIC_AWS_S3_*` |
| MinIO | `MINIO_*`, `NEXT_PUBLIC_MINIO_*` |

### Storage Priority

The system auto-detects storage based on available environment variables:
1. MinIO (if `NEXT_PUBLIC_MINIO_BUCKET` and `NEXT_PUBLIC_MINIO_DOMAIN` set)
2. Cloudflare R2
3. AWS S3
4. Vercel Blob (default)

Override with: `NEXT_PUBLIC_STORAGE_PREFERENCE`

## Database Configuration

### PostgreSQL

```bash
POSTGRES_URL=postgresql://user:pass@host:5432/db?sslmode=disable
DISABLE_POSTGRES_SSL=1
AUTH_TRUST_HOST=true
```

### Redis (Rate Limiting)

```bash
KV_URL=redis://redis:6379
# Or Upstash:
KV_REST_API_URL=https://xxx.upstash.io
KV_REST_API_TOKEN=xxx
```

## Docker Compose

### Development

```bash
docker compose -f dev-docker-compose.yml up
```

- Hot-reload via volume mounts
- Exposed PostgreSQL (5432)
- Exposed MinIO Console (9001)

### Production

```bash
docker compose up -d
```

- Built Docker image
- Persistent volumes
- Healthchecks

### Override Customization

```bash
cp docker-compose.override.yml.example docker-compose.override.yml
# Edit as needed
docker compose up -d
```

## Common Tasks

### Add new environment variable

1. Add to `.env.example.local` and `.env.example.production`
2. Document in README.md customization section
3. Add to `src/app/config.ts` if needed

### Add new storage provider

1. Create provider file in `src/platforms/storage/`
2. Export functions: put, copy, delete, list, getSignedUrl
3. Add to `src/platforms/storage/index.ts`
4. Add config checks in `src/app/config.ts`
5. Update storage selection logic

### Modify database schema

1. Prisma-style migrations not used
2. Direct SQL via `pg` client in `src/platforms/postgres.ts`
3. Models defined in `src/**/model.ts`

## Known Issues

1. **MinIO pre-signing** - Use `forcePathStyle: true` in S3 client (already implemented)
2. **Login redirect** - Always set `NEXTAUTH_URL` and `NEXT_PUBLIC_DOMAIN`
3. **Build requires DB** - PostgreSQL must be running during build
